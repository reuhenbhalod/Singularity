# Screenshots

The main [`README.md`](../../README.md) uses placeholder images so the repo
looks finished before real captures exist. When you are ready, take the four
screenshots below, save them here as PNGs with these exact names, and swap the
placeholder `src` values in the README for the local paths (for example
`src="docs/screenshots/shell.png"`).

Aim for a consistent look: the same wallpaper, the same appearance mode (dark
suits the shell), and a clean menu bar. The fullscreen shell is landscape, so
wide captures (around 1200 by 750, or a full Retina frame scaled down) read
best in the two-by-two grid.

| File name | What to capture |
|---|---|
| `shell.png` | The fullscreen shell right after summoning it: the command line with something typed, and the empty or lightly populated session log. |
| `panes.png` | Two or more results tiled as panes at once, to show the compositor (for example a video pane beside a mail pane). |
| `web.png` | A single web command running in place inside an embedded pane (the YouTube newest-video command is the most exercised path). |
| `settings.png` | The Settings window with its left-sidebar navigation, on a tab that shows off the app (Permissions or Safety works well). |

Once the four PNGs are here and the README `src` values point at them, delete
this note or leave it as a guide for future captures.
